# Task

把一个已经有多条正式路由的移动端 H5，从“长文档流页面集合”重构成统一壳层 + 分步骤主链路。

# Before

- 顶部标题栏和底部 Tab 看起来存在，但会跟着整页滚动
- 上传页同时承载上传准备、候选片段选择和微调
- 各页面边距、CTA、滚动方式不统一

# Goal

先统一移动端壳层，再把主链路拆成单任务页面，避免继续在一个页面里叠加多个阶段。

# Recommended structure

- 先做全局 shell：header 固定、main 独立滚动、tab 固定、safe-area 正确
- 再收敛上传页：只保留动作、文件、基础校验、进入粗扫
- 粗扫完成后进入独立选片页
- 处理中与错误页隐藏全局 Tab，保持单任务心智

# Key implementation notes

- 不要先局部修页面，再回头补壳层
- 不要让 `BottomCTA` 继续附着在长内容底部
- 候选片段选择与高级微调应拆层，不默认同层常驻
