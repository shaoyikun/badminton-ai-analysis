# Task

当页面交互是否合理带有主观性时，参考成熟开源项目或主流移动端产品的交互模式，而不是凭空设计。

# Before

- 需求只说“看起来更像产品”
- 页面结构和组件模式存在多种可能
- 纯靠主观判断容易做出不常见交互

# Goal

借鉴成熟模式中的信息层级和操作结构，让当前页面更稳妥，而不是发明新交互。

# Recommended structure

- 先找同类页面：表单页、上传页、结果页、列表页、错误恢复页
- 重点看信息层级、主操作位置、卡片分组和结果摘要方式
- 结合本仓库现有组件和品牌语境做适配
- 在总结里说明参考的是哪一类模式、为什么合适

# Key implementation notes

- 学“结构”和“心智”，不要机械抄视觉皮肤
- 优先参考成熟设计系统和主流移动端 App 的常见模式
- 如果参考模式和当前数据流冲突，先保留真实业务约束
- 说明参考依据时，讲清楚适配原因，不要只说“因为更好看”
