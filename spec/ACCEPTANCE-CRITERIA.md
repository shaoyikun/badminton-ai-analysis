# ACCEPTANCE-CRITERIA

## 1. 产品规格验收
- 主文档必须统一为主文档制，不保留无必要平行版本
- 所有产品定义以移动端首发为前提
- 主流程与异常流程清晰可追踪

## 2. 原型验收
- 必须存在完整主演示链路：
  首页 → 引导 → 规范 → 上传 → 确认 → 分析 → 完成 → 报告 → 复测
- 必须存在关键异常状态：
  上传失败、识别失败、多人同框、空历史记录
- 关键页面可在浏览器正常打开

## 3. MVP 技术验收（下一阶段）
- 能接收移动端上传视频
- 能创建和追踪分析任务
- 能返回结构化分析结果
- 能返回可分类的错误原因

## 4. 当前阶段结论
当前项目已基本满足产品规格与原型规格的前期验收，下一阶段可转入技术 PoC。
