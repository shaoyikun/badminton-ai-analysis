# INTERACTION-SPEC

## 1. 设计原则
- 移动端优先，默认竖屏单列布局
- 每个页面只承载一个核心任务
- 降低首次使用门槛
- 减少无效上传
- 报告结构先结论后细节
- 清晰告诉用户下一步做什么

## 2. 核心页面
- 首页
- 拍摄规范页
- 上传页
- 分析中页
- 报告页
- 历史记录页
- 复测对比页
- 错误状态页

说明：
- `首次引导页`、`上传确认页`、`分析完成页` 已从独立页面集合移除

## 3. 页面流转
- 主流程：首页 → 拍摄规范 → 上传 → 分析中 → 报告
- 辅助流程：首页 / 报告 → 历史记录 → 复测对比
- 异常流程：上传 / 分析中 → 错误状态页

## 4. 关键页面要求
### 首页
- 快速建立产品认知
- 首页先选择本次分析动作
- 清晰的“开始分析当前动作”入口
- 历史记录可作为次入口

### 上传页
- 动作类型选择
- 视频上传
- 基础信息预览
- 上传约束提示
- 上传前就绪检查
- 页内确认卡
- 点击确认后直接进入分析流程

### 分析中页
- 必须展示当前动作与阶段进度
- 必须展示当前任务摘要和分步骤实时说明
- 成功后直接进入报告页
- 失败后直接进入错误页

### 报告页
信息层级：
1. 动作名称 + 一句话结论 + 辅助总分
2. 当前最关键的问题及影响
3. 下次复测关注点
4. 维度分数与标准动作关键帧对比
5. 其余需要继续看的问题
6. 再次上传 / 历史记录 / 复测入口

### 历史记录页
信息层级：
1. 已有同动作历史样本数量
2. 为什么要继续复测
3. 历史样本列表
4. 历史详情
5. 在历史详情中一键设为当前对比基线

### 复测对比页
信息层级：
1. 本次整体变化结论
2. 最明显的改善项
3. 最该防止继续回落的点
4. 下次复测只先盯什么

### 错误状态页
- 必须说明错误原因
- 必须解释为什么会失败
- 必须给出重拍或重传建议
- 必须有返回拍摄规范或重新上传入口
- 返回上传页后必须能看到上次失败摘要，但需要重新选择文件

## 5. 用户可见状态映射
- `created`：待上传
- `uploaded` + 预处理阶段：上传中 / 校验中
- `processing`：分析中
- `completed`：已完成
- `failed`：失败

## 6. 错误映射约束
- `upload_failed` / `invalid_duration`：优先回上传
- `multi_person_detected` / `body_not_detected` / `poor_lighting_or_occlusion` / `invalid_camera_angle`：优先回拍摄规范
- `preprocess_failed` / `pose_failed`：允许重新上传，也允许回看拍摄规范
