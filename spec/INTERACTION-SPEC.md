# INTERACTION-SPEC

## 1. 设计原则
- 移动端优先，默认竖屏单列布局
- 每个页面尽量只承载一个核心任务
- 降低首次使用门槛
- 减少无效上传
- 报告结构一眼看懂
- 清晰告诉用户下一步做什么

## 2. 核心页面
- 首页
- 首次引导页
- 拍摄规范页
- 上传页
- 上传确认页
- 分析中页
- 分析完成页
- 报告页
- 历史记录页
- 复测对比页
- 错误状态页

## 3. 关键页面要求
### 首页
- 快速建立产品认知
- 清晰的“开始分析”入口

### 上传页
- 动作类型选择
- 视频上传
- 基础信息预览
- 进入分析前确认

### 报告页
信息层级：
1. 诊断摘要 + 一句话结论
2. 当前最关键的问题
3. 历史记录 / 复测入口
4. 复测对比结论
5. 标准动作关键帧对比
6. 其余需要继续看的问题
7. 下次复测关注点

### 历史记录页
信息层级：
1. 已有同动作历史样本数量
2. 为什么要继续复测
3. 历史样本列表
4. 手动选择对比基线入口

### 复测对比页
信息层级：
1. 本次整体变化结论
2. 最明显的改善项
3. 最该防止继续回落的点
4. 下次复测只先盯什么

### 错误状态页
- 必须说明错误原因
- 必须给出重拍建议
- 必须有返回拍摄规范或重新上传入口
