# INTERACTION-SPEC

## 1. 设计原则
- 移动端优先，默认竖屏单列布局
- 每个页面尽量只承载一个核心任务
- 降低首次使用门槛
- 减少无效上传
- 报告结构一眼看懂
- 清晰告诉用户下一步做什么

## 2. 核心页面
- 首页
- 首次引导页
- 拍摄规范页
- 上传页
- 上传确认页
- 分析中页
- 分析完成页
- 报告页
- 历史记录页
- 复测对比页
- 错误状态页

## 3. 关键页面要求
### 首页
- 快速建立产品认知
- 清晰的“开始分析”入口

### 上传页
- 动作类型选择
- 视频上传
- 基础信息预览
- 进入分析前确认

### 报告页
信息层级：
1. 总分 + 一句话结论
2. Top 3 问题
3. 训练建议
4. 关键帧对比
5. 复测建议

### 错误状态页
- 必须说明错误原因
- 必须给出重拍建议
- 必须有返回拍摄规范或重新上传入口
