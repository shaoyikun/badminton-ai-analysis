# 羽毛球 AI 动作分析产品 PRD V1

## 1. 文档信息
- 项目名称：羽毛球 AI 动作分析与复测反馈平台
- 文档版本：V1.1
- 文档类型：产品需求文档（PRD）
- 更新时间：2026-03-12
- 当前状态：MVP 产品交互基线

---

## 2. 产品概述

### 2.1 产品一句话
用户在移动端上传一段羽毛球动作视频，系统返回动作问题诊断、问题影响说明、标准动作差异和下次复测关注点。

### 2.2 首发平台约束
- 产品首发阶段优先面向移动端上线
- 默认以手机竖屏场景为核心使用场景
- 后续交互设计、页面结构、技术实现与 Demo 展示，均以移动端优先为标准

### 2.3 产品目标
本产品希望解决羽毛球爱好者“看不懂自己动作问题、没有持续反馈、缺少低成本复盘工具”的问题。

### 2.4 核心价值
1. 帮用户识别当前最关键的动作问题
2. 帮用户理解问题会如何影响击球结果
3. 帮用户看懂和标准动作、过去自己的差异
4. 帮用户形成训练—复测—进步跟踪闭环

---

## 3. 用户与场景

### 3.1 目标用户
#### 核心用户
- 每周有固定打球频率的羽毛球爱好者
- 想提升动作质量但缺少教练长期陪练的人
- 正在上私教课，希望课后复盘的人

#### 潜在扩展用户
- 羽毛球教练
- 球馆 / 俱乐部运营者
- 青训机构

### 3.2 用户痛点
- 不知道自己动作哪里错
- 录了视频也看不出来关键问题
- 线下教练成本高、反馈频率低
- 练习后无法量化自己是否进步

### 3.3 核心使用场景
1. 自主训练后上传视频，获取动作诊断
2. 私教课后上传视频，验证是否掌握动作要点
3. 一段时间后复测，对比动作是否改进

---

## 4. MVP 产品范围

### 4.1 支持动作
MVP 首期仅支持：
1. 正手高远球
2. 杀球

### 4.2 输入约束
- 单人视频
- 5~15 秒
- 固定机位
- 推荐侧后方 / 正后方视角
- 一段视频仅分析一种动作

### 4.3 输出结果
- 动作类型
- 一句话结论
- 总评分（辅助信息，不作为页面主叙事）
- 分维度评分
- Top 问题
- 问题影响说明
- 标准动作关键帧对比
- 复测反馈
- 下次复测关注点 / 轻量改进建议

### 4.4 MVP 不做
- 实时动作指导
- 多动作自动混合识别
- 复杂比赛视频分析
- 多人同场复杂识别
- 战术级分析
- 完整教练后台
- 结构化训练计划、练习组数与课程编排

---

## 5. 产品目标与成功标准

### 5.1 MVP 要验证的关键问题
1. 用户是否愿意上传自己的训练视频
2. 用户是否觉得分析结果“说到点上”
3. 用户是否愿意基于问题反馈继续复测
4. 用户是否愿意为更详细报告或训练服务付费

### 5.2 核心指标
#### 产品指标
- 视频上传完成率
- 有效视频识别率
- 报告查看完成率
- 二次上传率
- 复测率
- 用户主观认可度

#### 商业指标
- 免费转付费率
- 单次报告购买率
- 订阅转化率
- 教练试用意愿

## 6. MVP 页面架构

MVP 页面集合统一收口为 8 页：

1. 首页
2. 拍摄指引页
3. 上传页
4. 分析中页
5. 报告页
6. 历史记录页
7. 复测对比页
8. 错误状态页

说明：
- `首次引导页` 不再作为独立页面，内容并入首页与拍摄指引页
- `上传确认页` 不再作为独立页面，确认信息内嵌在上传页
- `分析完成页` 不再作为独立页面，分析成功后直接进入报告页

---

## 7. 用户流程

### 7.1 主流程
首页 → 拍摄指引 → 上传 → 分析中 → 报告 → 历史记录 / 复测对比

### 7.2 辅助流程
- 首页 → 历史记录
- 报告页 → 历史记录
- 报告页 → 复测对比
- 历史记录 → 历史详情 → 设为对比基线 → 复测对比

### 7.3 异常流程
- 上传前校验失败 → 留在上传页做页内提示
- 服务端返回明确错误 → 进入错误状态页
- 识别失败 → 提示调整机位 / 光线 / 视频长度后重新上传

---

## 8. 功能需求拆解

### 8.1 首页
#### 目标
承接首次进入用户，快速告诉用户“这是什么、怎么用、支持什么动作”。

#### 功能点
- 产品价值说明
- 支持动作展示
- 3 步使用说明
- 开始分析入口
- 历史记录入口

#### 优先级
P0

### 8.2 拍摄指引
#### 目标
提升可识别率，降低无效上传。

#### 功能点
- 推荐拍摄角度说明
- 推荐距离说明
- 视频时长与人数要求
- 常见错误示例
- 动作专项提醒
- 去上传按钮

#### 优先级
P0

### 8.3 视频上传
#### 目标
完成动作选择、文件选择、上传前校验和启动分析。

#### 功能点
- 选择动作类型
- 上传视频
- 展示视频预览
- 展示视频基础信息
- 页内展示拍摄确认信息
- 点击后直接进入分析流程

#### 交互规则
- 未上传视频时，点击分析按钮仅做页内提示
- 切换动作类型且已有文件时，清空当前视频并提示重新选择
- 上传失败优先使用页内提示；仅在服务端返回明确错误码时进入错误页

#### 优先级
P0

### 8.4 分析中状态
#### 目标
降低等待焦虑，让用户理解系统正在处理什么。

#### 功能点
- 分析进度文案
- 预计耗时提示
- 状态切换：视频已上传 / 正在校验与抽帧 / 正在识别动作特征 / 正在生成诊断与复测建议
- 成功后自动进入报告
- 失败后自动进入错误页

#### 优先级
P0

### 8.5 分析报告页
#### 目标
把诊断结果清晰传递给用户，并明确这次最值得优先关注的问题与下次复测重点。

#### 功能点
- 顶部摘要
- 一句话结论
- 辅助总分
- 当前最关键问题及影响
- 其余问题
- 标准动作关键帧对比
- 下次复测关注点
- 再次上传 / 历史记录 / 复测对比入口

#### 信息层级
1. 动作名称 + 一句话结论 + 辅助总分
2. 当前最关键问题及影响
3. 下次复测关注点
4. 维度分数与标准动作差异
5. 继续动作入口

#### 优先级
P0

### 8.6 历史记录页
#### 目标
承接持续复测，而不是仅做结果归档。

#### 功能点
- 同动作历史记录列表
- 样本数量 / 最近趋势提示
- 单次历史样本详情
- 历史详情中的标准动作差异摘要
- 历史详情中的下次复测关注点
- 在历史详情中设为当前对比基线

#### 优先级
P1

### 8.7 复测对比
#### 目标
体现“进步”而不是“单次诊断”。

#### 功能点
- 最近两次结果对比
- 关键维度提升 / 下降提示
- 最需防回落的点
- 下次复测重点提示
- 支持从历史详情直接进入当前复测对比

#### 约束
- 仅允许同动作类型样本进入复测对比
- 仅允许已完成样本作为对比基线

#### 优先级
P1

### 8.8 错误状态页
#### 目标
说明失败原因，并给出明确的下一步。

#### 功能点
- 错误标题
- 失败原因说明
- 重拍 / 重传建议
- 返回拍摄指引入口
- 重新上传入口

#### 优先级
P0

## 9. 页面字段与状态规则

### 9.1 页面字段清单
#### 首页
- 产品标题
- 产品副标题
- 支持动作列表
- 使用说明
- 开始分析按钮
- 历史记录入口

#### 上传页
- 动作类型
- 文件上传控件
- 视频预览
- 视频时长 / 文件大小
- 规范提醒
- 开始分析按钮

#### 报告页
- 一句话结论
- 总分（辅助）
- 维度分数
- 核心问题
- 影响说明
- 下次复测关注点
- 复测建议

### 9.2 用户可见状态映射
- 待上传：尚未选择文件或尚未启动分析
- 上传中 / 校验中：文件上传和预处理阶段
- 分析中：动作特征识别与报告生成阶段
- 已完成：报告已可查看
- 失败：需要重拍、重传或查看拍摄规范

### 9.3 错误类型范围
- 上传失败
- 时长不符合要求
- 多人同框
- 人体未识别
- 光线 / 遮挡问题
- 机位不合适
- 预处理失败
- 姿态识别失败

---

## 10. 产品规则
- 每次只分析一种动作
- 不符合拍摄规范的视频应优先拒绝，而不是给低质量结果
- 报告必须聚焦最重要的 3 个问题，避免信息过载
- 复测关注点必须落到具体观察维度，不能只给抽象评价
- 总评分只能作为辅助摘要，不能抢占主要信息层级
- 历史记录和复测对比只面向同动作、已完成样本

---

## 11. 非功能需求
- 报告语言清晰、非专业用户可理解
- 等待时间可接受，并有明确反馈
- 用户视频与分析结果需支持删除
- 不得默认公开用户视频内容

---

## 12. 商业化假设

- 付费解锁更详细报告 / 多次上传 / 历史分析 / 更强的复测反馈

### B 端（后续）

- 球馆 / 俱乐部学员成长档案
