# 羽毛球 AI 动作分析项目技术方案 V0

## 1. 目标

本方案用于支撑 MVP 阶段的产品能力：
- 用户上传羽毛球动作视频
- 系统识别动作类型与关键动作阶段
- 与标准动作模板进行对比
- 输出动作问题与训练建议

当前阶段分为两层：
1. Demo 层：前端模拟分析，先验证产品体验
2. MVP 层：接入真实视频分析与姿态识别能力

---

## 2. 当前 Demo 技术形态

当前已实现一个轻量前端 demo：
- 静态网页
- 本地视频上传与预览
- 模拟分析中状态
- 针对“正手高远球 / 杀球”输出不同的模拟报告

### 优势
- 开发快，便于演示
- 先验证交互链路与报告结构
- 后续可逐步替换为真实模型输出

### 当前不足
- 未接真实模型
- 未做后端存储
- 未做真实姿态识别和关键点抽取

---

## 3. MVP 建议技术架构

## 3.1 整体架构

前端 Web / 小程序 / App
→ 上传视频
→ 后端 API
→ 视频预处理服务
→ 姿态识别服务
→ 动作评估服务
→ 报告生成服务
→ 返回结果给前端

---

## 3.2 模块拆解

### A. 前端层
负责：
- 动作类型选择
- 视频上传
- 拍摄规范展示
- 分析中状态展示
- 报告页展示
- 历史记录查看

### B. API 层
负责：
- 接收上传请求
- 创建分析任务
- 查询分析状态
- 返回分析报告

### C. 视频预处理层
负责：
- 视频格式标准化
- 抽帧
- 关键片段切分
- 检测是否单人、是否遮挡严重

### D. 姿态识别层
负责：
- 提取人体关键点
- 跟踪动作过程中的关键关节位置
- 提取挥拍动作时序信息

### E. 动作评估层
负责：
- 判断动作阶段
- 计算与标准模板的差异
- 输出维度评分
- 识别 Top 问题

### F. 报告生成层
负责：
- 将结构化分析结果转成人类可读报告
- 输出训练建议与复测建议

---

## 4. 姿态识别与动作分析建议路线

## 4.1 初期可选方案

### 方案 1：MediaPipe / MoveNet
优点：
- 上手快
- 成熟稳定
- 适合先做 2D 关键点识别

缺点：
- 在高速挥拍、遮挡场景下表现可能一般

### 方案 2：YOLO Pose / ViTPose
优点：
- 灵活性更高
- 可扩展性更强

缺点：
- 训练和调参成本更高

### 初步建议
MVP 阶段先用成熟姿态估计方案快速起步，再逐步积累专项数据。

---

## 5. 动作评估逻辑建议

MVP 阶段不要做复杂端到端判断，而采用“规则 + 模型”混合方案。

### 评估维度示例
- 准备姿态
- 引拍完整度
- 转体 / 转髋
- 击球点位置
- 跟随动作
- 回位意识

### 流程
1. 从关键点序列中提取动作特征
2. 根据规则判断关键问题
3. 用模板差异辅助打分
4. 由建议生成模块输出结果

---

## 6. 报告生成建议

建议不要完全交给大语言模型自由生成，而采用：

- 结构化判断结果
- 问题标签体系
- 建议模板库
- LLM 仅负责润色与用户表达

这样可以减少“看起来很像对，其实不专业”的风险。

---

## 7. 数据需求

MVP 阶段建议优先积累：
- 标准动作示范视频
- 用户练习视频
- 动作问题标签
- 问题到训练建议映射
- 教练点评样本

---

## 8. 技术演进路线

### Stage 1：可演示 Demo
- 完成前端交互 demo
- 验证报告结构与用户体验

### Stage 2：半自动 MVP
- 接入真实视频上传
- 接入姿态识别
- 用规则生成评分与建议

### Stage 3：增强型 MVP
- 增加历史记录
- 增加复测对比
- 增加更多动作类型

### Stage 4：服务化版本
- 支持教练后台
- 支持学员管理
- 支持课后报告自动生成

---

## 9. 当前建议

从今天起，技术推进顺序建议是：
1. 先把 demo 体验打磨到可展示
2. 再补拍摄规范与报告模板
3. 然后开始接真实姿态识别能力
4. 最后再考虑更复杂的模型和多动作扩展
