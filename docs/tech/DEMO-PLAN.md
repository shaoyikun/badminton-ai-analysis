# Demo 实现计划

## 今日目标

做出一个可用的产品 demo，至少具备：

1. 选择动作类型
2. 上传本地视频
3. 展示分析中状态
4. 输出结构化分析报告
5. 能体现后续真实产品“问题诊断 + 复测反馈”的核心价值

## Demo 方案

本阶段先做前端交互 demo，不接真实 AI 模型。

### 为什么这样做
- 今天可以快速形成可演示成果
- 先验证产品体验、问题表达和复测反馈结构
- 后续可以把“模拟分析结果”逐步替换为真实模型输出

## Demo 范围
- 单页 Web Demo
- 支持正手高远球 / 杀球
- 支持上传本地视频并预览
- 支持进入“分析中”状态
- 输出一份模拟分析报告
- 支持切换不同动作的不同报告结果

## 后续可替换模块
- 视频上传接口
- 姿态识别服务
- 动作评估逻辑
- 报告生成逻辑
